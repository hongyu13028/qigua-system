# 历史解读 —归档目录

>用途：把"问卦系统 v0.x期间对黄总实际业务的解读记录"归档到这里，**不进入 agent memory / 新会话上下文**。
> 黄总明确要求："新开会话不会受到上下文的影响"——所以业务历史全部落盘，需要时按文件名检索。

## 为什么建这个目录

之前的业务历史（智顺系列、1688店铺定位、潜客话术、起卦结论等）都散落在 agent memory 里，
新会话一开会就自动注入——黄总不希望这样。
清理方案：

| 类型 |归宿 |
|---|---|
| 元信息（协作风格、网络环境、API key 等） |留在 agent memory |
|业务历史（具体某次问卦解读、某次定位、某次话术） |归档到本目录 |
|问卦基础设施（仓库路径、CLI 用法、prompt位置） |已在 `qigua_prompt.md` / `使用说明.md` 里 |

##检索方式

新会话需要查历史时：
1. `Read D:\问卦系统\历史解读\README.md` 看索引
2. 按日期/主题打开对应文件
3. 不要自动注入到 system prompt——只是当黄总主动问到时再读

##维护规则

- 黄总每跑完一次"对真实业务的问卦/解读"，归档一份到此目录
- 文件命名：`YYYY-MM-DD-主题-一句话.md`
- 内容包含：起卦参数、卦象、解卦要点、行动建议、应期、复盘结果
-同步 git push 到 Gitee（保留黄总双备份习惯）
