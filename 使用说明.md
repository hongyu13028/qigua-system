# 问卦系统 v0.1 — 使用手册

> 基于头条 @灿若星汉《我把《易经》64卦编成了人生通关攻略》改编的实战决策辅助系统

---

## 这是个什么东西

把《易经》64 卦的"决策智慧"提炼出来，配合 AI，做成一套**结构化思考 + 行动建议**的轻量决策辅助。

**核心定位**：
- ❌ 不是算命
- ❌ 不是预测
- ❌ 不是替你做决定
- ✅ 是给你一个"框架", 让你在乱局中看见自己看不见的那一面
- ✅ 是用古人的智慧 + AI 的能力, 输出"宜/忌"明确的行动建议

---

## 包含什么

| 文件 | 作用 |
|---|---|
| `gua_64.json` | 64 卦数据库（卦名、上下卦、二进制、模块、主题、判断、现代解读） |
| `qigua_prompt.md` | 问卦先生 System Prompt（贴进任何 LLM 就能用） |
| `qigua.py` | 起卦工具（数字/时间/铜钱三种起卦方式，CLI） |
| `文章学习笔记.md` | 那篇头条文章的结构化学习笔记 |
| `使用说明.md` | 本文件 |

## 三种使用姿势

### 姿势 A：轻量（5 分钟）
**直接用 Python 脚本起卦 + 看骨架**

```bash
python qigua.py
# 跟着提示选 1/2/3，输入数字或爻位
# 输出：本卦 + 之卦 + 互卦 + 错卦 + 综卦 + 上下关系
```

适合：
- 心里有个具体问题想快速问
- 想要一个"卦象坐标", 知道问题落在哪个模块
- 急着要行动建议

---

### 姿势 B：中等（10-15 分钟）
**CLI 起卦 + LLM 深度解读**

1. 跑 `python qigua.py --demo` 或自己起卦，得到卦象报告
2. 把 `qigua_prompt.md` 整个内容贴进 LLM 的 System Prompt 框
3. 在对话框里：把 CLI 输出 + 你的问题一起贴给 LLM
4. LLM 给出"宜/忌/应期"完整解读

适合：
- 重要决策（融资、换合伙人、战略调整）
- 想多角度解卦
- 想要可复盘的结构化建议

---

### 姿势 C：深度（30 分钟+）
**用 Mavis 团队模式做完整解卦 + 复盘**

1. 起卦 → 拿骨架
2. 调用 Mavis Team 的 `deep-research` skill
3. 输入：卦象 + 问题 + 你的处境
4. Team 会调 3 个角色：解卦先生（古文派）、商业顾问（落地派）、心理教练（情绪派）
5. 三方辩论 → 综合建议 → 给你一个完整的决策方案

适合：
- 重大人生/商业决策
- 心里焦虑需要多角度印证
- 想用 AI 团队的力量

## 问卦的正确姿势

### ✅ 好的问法
- "我这个 1688 店铺下个月能否出 10 单？"
- "和这位合伙人的合作能否持续 6 个月以上？"
- "我是否应该在这个时间点采购 XX 原料？"
- "目前这个产品定位, 客户是否会买单？"
- "下周一去和客户谈判, 这次能成吗？"

**共同特点**：
- 一件事一卦
- 时间窗口明确
- 可以判吉凶
- 不情绪化

### ❌ 错误问法
- "我今年能不能发财？"（范围太大）
- "我喜欢的男生不喜欢我, 怎么让他喜欢我？"（不可执行）
- "我下周能不能签下那个大客户？"（同时问了多件事）
- "我该不该继续做 1688？"（问题边界不清）

**AI 怎么应对**：
- 对模糊问题, 必须追问到清晰才起卦
- 对情绪化问题, 先安抚, 再引导改写
- 拒绝"求神问卦"型问题, 一律改写为决策型问题

## 实战案例：5 步问卦

**场景**：黄总 1688 店铺, 财务危机临界, 想问"我下一步最该做什么？"

**第 1 步：定模块** → 模块 4 立德修身（历经世事沧桑，回归内在修炼）

**第 2 步：起卦** → 数字 88, 18
- a=88 → 上卦 88%8=0 → 坤
- b=18 → 下卦 18%8=2 → 兑
- (88+18)%6 = 106%6 = 4 → 变爻在第四爻
- 本卦 = 上坤下兑 = 临卦 ䷒
- 之卦 = 第四爻阳变阴 = 地泽临 → 山泽损
- 互卦 = 下卦[2,3,4] + 上卦[3,4,5] = 兑+坤 = 萃卦
- 错卦 = 全反 = 雷水解
- 综卦 = 上下颠倒 = 地泽临(本身)

**第 3 步：解卦** (节选 AI 输出)：
> 临卦：亲临督导, 临近。下兑上坤, 上生下 (土生金, 比和)。形势对你托底。
> 之卦损：减损的平衡艺术。该损的损, 把资源往核心业务上聚。
> 互卦萃：聚集人心。下一步是"凝团队, 聚资源"。

**第 4 步：行动**
- ✅ 宜：聚焦核心 SKU, 砍掉边缘品
- ✅ 宜：和团队深谈一次, 统一目标
- ❌ 忌：盲目铺新品类
- ❌ 忌：硬撑不赚钱的链接
- ⏰ 应期：30 天后复盘

**第 5 步：复盘**
- 30 天后再起一卦, 看是否从"临"走到"损"或"泰"
- 记录在 `gua_log.md` 里, 形成自己的"卦象档案"

## 常见问题

**Q1: AI 起卦准吗？**
A: 起卦本身是符号化工具, 准不准看你怎么用。把它当"决策框架"用, 比当"预言"用价值大得多。

**Q2: 同一件事能反复起卦吗？**
A: 古训是"一事一卦", 事情没新变量, 别重复问。但现实事件进入新阶段了, 可以重新起卦。

**Q3: 怎么判断"是不是新变量"？**
A: 四问判断法：
- 对象是不是同一个（人/项目/机会）
- 决策点是不是同一个
- 时间窗口是不是同一阶段
- 现实里有没有新动作/通知/回复/合同
四件都没变, 就是同题, 不要再起。

**Q4: 这套系统会替代我思考吗？**
A: 不会。AI 给你的是结构, 不是答案。**卦象是镜子, 不是算盘**。最终拍板的还是你自己。

**Q5: 怎么开始？**
A: 现在就找一个具体的事, 跑 `python qigua.py`, 跟着流程走一遍。第一次可能很怪, 第三次就会上瘾。
